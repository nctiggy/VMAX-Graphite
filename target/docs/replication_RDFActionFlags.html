occurs		 description
all (boolean) 1/1 Lists all SRDF mirrors of the selected devices. Used with symrdf list. When performing an SRDF control or set operation, it targets the SRDF action at all devices in the device group: Standard SRDF devices and locally-attached BCV SRDF devices. This option is only supported for list and device group operations.
bcv (boolean) 1/1 Targets the SRDF action at the Device Group's locally-associated BCV volumes that are configured as SRDF BCV volumes.
bypass (boolean) 1/1 Bypasses any existing Symmetrix exclusive locks during an SRDF operation. WARNING: Only use this flag if you are certain no other SRDF operation is in progress at the local and/or remote Symmetrix arrays.
full (boolean) 1/1 Requests a full establish or restore operation. By default, if the -full option is not specified, an incremental establish or restore is attempted.
hop2 (boolean) 1/1 Targets the SRDF action at the group's second-hop volumes in a cascaded SRDF relationship. For example, in an RDF1 group, the action targets the R21->R2 pair of the R1->R21->R2 relationship.
remote (boolean) 1/1 Requests a remote data copy flag with failback, failover, restore, update, and resume. When the concurrent link is ready, data is copied to the concurrent SRDF mirror. These operations require the remote data copy option, or the concurrent link to be suspended.
immediate (boolean) 1/1 Applies only to SRDF/A-capable devices. Causes failover, split, and suspend commands to drop the SRDF/A session immediately.
consExempt (boolean) 1/1 Allows devices to be added, removed, or suspended without affecting the state of the SRDF/A session or requiring that other devices in the session be suspended. Used for an SRDF group supporting an active SRDF/A session. When used with list operations, lists devices that are consistency exempt or that are paired with devices that are consistency exempt.
rp (boolean) 1/1 Allows the operation even though one or more volumes are tagged for RecoverPoint. When used with refresh, only allowed for refresh R1.
establish (boolean) 1/1 Begins a volume copy. When used with createpair, a full copy from the source to the target is started, synchronizing the dynamic SRDF pairs in the volume file. When used with failover, the dynamic SRDF volume pairs swap personality and an incremental establish is initiated.
restore (boolean) 1/1 Begins a full copy from the target to the source, synchronizing the dynamic SRDF pairs in the volume file. When used with failover, the dynamic SRDF volume pairs swap personality and an incremental restore is initiated.
refreshR1 (boolean) 1/1 Marks the source (R1) volumes to refresh from the remote mirror.
refreshR2 (boolean) 1/1 Marks the target (R2) volumes to refresh from the remote mirror.
force (boolean) 1/1 Attempts to force the operation even though one or more volumes in the Device Group may not be in the normal, expected SRDF state or SRDF mode for that operation.
symForce (boolean) 1/1 Requests the Symmetrix array force operation be executed when normally it is rejected. Use extreme caution when using this option. CAUTION: Use care when applying -symforce, as data could be lost or corrupted. Use of this option is not recommended, except in an emergency. NOTE: To enable -symforce, a parameter called SYMAPI_ALLOW_RDF_SYMFORCE in the options file must be set to TRUE. When used with -symforce, a split command executes on an SRDF pair, even when the pair is sync in progress or restore in progress. During the execution of an establish or restore command, -symforce prohibits the verification of valid tracks on the device at the source.
star (boolean) 1/1 Specifies the action is targeted for volumes in STAR mode.
halfSwap (boolean) 1/1 Swaps the SRDF personality of one-half of the designated dynamic SRDF pair. Source R1 devices become target R2 devices, and target R2 devices become source R1 devices.
